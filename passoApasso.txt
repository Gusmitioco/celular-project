ConSERT FÁCIL — Como rodar (Linux)

PORTAS
- Postgres: 5432
- Backend: 3001
- Frontend: 3000

========================================
1) RODAR NA SUA MÁQUINA (clone/zip do projeto)
========================================

1.1) Entrar na raiz do projeto
cd ~/consert-facil

1.2) Subir o banco (Docker)
docker compose up -d db

1.3) Backend (primeira vez / sempre que mudar o schema)
cd backend
cp .env.example .env   (edite se precisar)
npm install
npm run prisma:generate
npm run prisma:migrate   (quando pedir nome: digite "init")
npm run prisma:seed

1.4) Rodar o backend
npm run dev

1.5) Frontend
cd ../frontend
cp .env.example .env.local   (edite se precisar)
npm install
npm run dev

1.6) Testes rápidos
curl http://localhost:3001/health
curl http://localhost:3001/api/marcas
Abra: http://localhost:3000

========================================
2) RODAR EM OUTRA MÁQUINA VIA .ZIP (com dependências já instaladas no PC)
========================================

Pré-requisitos nessa outra máquina:
- Node 24 + npm
- Docker + Docker Compose
(As dependências do projeto NÃO precisam estar dentro do zip, mas npm install ainda é recomendado.)

2.1) Extrair o zip
unzip consert-facil.zip -d ~/consert-facil
cd ~/consert-facil

2.2) Subir banco
docker compose up -d db

2.3) Backend (garantir env e Prisma)
cd backend
cp .env.example .env
npm install
npm run prisma:generate
npm run prisma:migrate
npm run prisma:seed
npm run dev

2.4) Frontend
cd ../frontend
cp .env.example .env.local
npm install
npm run dev

========================================
3) SE QUISER “RODAR SEM npm install” (não recomendado)
========================================
Isso só funciona se você zipar com:
- backend/node_modules
- frontend/node_modules
E na nova máquina o SO/arquitetura for compatível.

Mesmo assim, ainda pode falhar por binários nativos.
O caminho seguro é sempre:
npm install

========================================
4) PARAR TUDO
========================================
- Frontend/backend: Ctrl+C no terminal de cada um
- Banco:
cd ~/consert-facil
docker compose down
(dados ficam salvos no volume; para apagar tudo: docker compose down -v)
